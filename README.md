# LibManager-byQt 图书管理系统

LibManager-byQt 是一个基于 **Qt 6 / QML / C++ / SQLite** 的桌面端图书管理系统。项目以 QML 构建交互界面，以 C++ 封装数据模型、业务服务、数据库访问和 QML 桥接层，面向普通读者与系统管理员提供登录注册、图书检索、单册借还、借阅流水、读者管理与统计排行等功能。

> 默认启动时，系统会初始化本地 SQLite 数据库 `LibManager.db`，并在数据库中不存在初始管理员时注入管理员账号：`10000`，密码：`MySerenades`。

## 总体功能模块

系统按角色和业务边界可划分为以下模块：

1. **界面交互模块**
   - `Main.qml` 提供读者登录、管理员登录和读者注册入口。
   - `Dashboard.qml` 提供登录后的主导航、分页列表、图书业务、借阅管理、读者业务和系统设置界面。

2. **QML-C++ 桥接模块**
   - `SystemBridge` 作为 `QML_SINGLETON` 暴露给前端。
   - 负责将 QML 的字符串、数字、布尔值等输入转换为后端实体对象。
   - 将后端 `Book`、`Volume`、`LoanRecord`、`ReaderAccount` 等对象打包为 `QVariantMap` / `QVariantList` 供 QML 绑定。

3. **用户与权限模块**
   - 支持普通读者登录、管理员登录、读者注册。
   - 支持读者修改用户名和密码。
   - 管理员拥有图书维护、单册维护、全局借阅流水查询和读者管理权限。

4. **图书与馆藏模块**
   - 支持按关键词检索图书。
   - 支持按书名、作者、出版社、ISBN 组合高级检索。
   - 支持图书详情查看、热门图书排行。
   - 管理员可新增/更新/软删除图书，新增/更新/删除单册。

5. **借阅流通模块**
   - 读者可借阅可用单册。
   - 管理员可归还单册。
   - 系统记录借阅日期、应还日期、归还日期、是否归还和是否逾期。

6. **统计分析模块**
   - 支持热门图书排行。
   - 支持借阅达人排行。
   - 支持读者按月/按年借阅历史和借阅数量统计。
   - 支持未归还、逾期数量统计。

7. **数据持久化模块**
   - SQLite 数据库自动建表。
   - DAO 层封装账户、图书、单册、位置、作者、借阅流水的读写。
   - 使用软删除字段 `IsValid` 保留历史关联数据。

## 详细功能模块

### 1. 登录、注册与会话状态

- 读者 ID 必须是 **8 位非零开头纯数字**。
- 管理员 ID 必须是 **5 位非零开头纯数字**。
- 密码长度必须不少于 8 位。
- 密码不会明文入库，而是采用随机盐加 SHA3-512 哈希后存储。
- 登录成功后，`SystemBridge` 保存当前读者或管理员状态，并向 QML 发出 `userInfoChanged` 信号刷新界面。
- `logout()` 会清除当前会话状态。

### 2. 读者功能

普通读者登录后可使用：

- 图书检索：通过关键词查询图书。
- 高级检索：按书名、作者、出版社、ISBN 等条件查询。
- 热门图书：查看借阅次数排名靠前的书籍。
- 图书详情：查看 ISBN、书名、作者、出版社、年份、分类、语言、简介和单册列表。
- 单册借阅：选择可借单册后提交借阅。
- 借阅历史：查看自己的借阅记录、借阅日期、应还日期、归还日期和状态。
- 高级流水检索：按 ISBN、单册号、状态和日期范围筛选自己的借阅流水。
- 账户资料：查看当前账号、姓名、权限、借阅上限、已借数量和逾期数量。
- 修改信息：修改用户名或密码。

### 3. 管理员功能

管理员登录后可使用：

- 图书检索、高级检索、热门图书。
- 图书维护：新增或更新图书元数据，包括 ISBN、题名、作者、出版社、出版年份、分类、语种、简介。
- 图书删除：对图书执行软删除。
- 单册管理：新增或更新单册编号、馆藏位置、可用状态、是否开架、备注。
- 单册删除：对单册执行删除/软删除类操作。
- 单册归还：对处于外借状态的单册进行归还处理。
- 全局流水查询：查看所有读者的借阅流水。
- 借阅高级检索：按 ISBN、单册号、借阅状态和时间范围查询流水。
- 读者管理：检索读者、查看读者信息、修改读者姓名、启用/禁用读者账户。

### 4. 数据库初始化与默认管理员

程序入口会先初始化数据库。初始化成功后才加载 QML 界面。首次启动时，若账户表中不存在 ID 为 `10000` 的管理员，系统会创建默认管理员。

### 5. 界面与导航

`Dashboard.qml` 按顶部菜单与子菜单组织功能：

- 主页
- 图书业务
  - 图书检索
  - 高级检索
  - 热门图书
  - 单册管理（管理员）
- 借阅管理
  - 借阅历史（读者）/ 流水查询（管理员）
  - 高级检索
  - 借阅统计（读者）
- 读者业务
  - 账户资料
  - 修改信息（读者）/ 读者管理（管理员）
- 系统设置
  - 界面偏好
  - 数据库连接状态
  - 退出系统

## 系统架构

系统采用较典型的分层结构：

```text
QML UI 层
  Main.qml / Dashboard.qml
        │
        ▼
桥接层
  SystemBridge（QML 单例、Q_INVOKABLE 接口、状态属性）
        │
        ▼
业务服务层
  UserService / LibraryService / StatService
        │
        ▼
DAO 数据访问层
  AccountDAO / BookDAO / LoanRecordDAO / DatabaseController
        │
        ▼
SQLite 数据库
  Account / Book / Authors / Volume / VolLocation / LoanRecord
```

### 分层职责

| 层级 | 主要文件 | 职责 |
|---|---|---|
| UI 层 | `Main.qml`, `Dashboard.qml` | 展示界面、用户输入、调用桥接接口、列表分页 |
| 桥接层 | `SystemBridge.h/.cpp` | 暴露 QML 可调用 API，维护登录状态，转换前后端数据结构 |
| 业务层 | `UserService.cpp`, `LibraryService.cpp`, `StatService.cpp` | 实现登录、注册、借阅、归还、维护、统计等业务规则 |
| 数据层 | `DatabaseManager.h` | 建库建表、DAO 查询、DAO 写入、事务和 SQL 异常处理 |
| 类型层 | `TypeCode.h`, `DataTemplates.h` | 定义枚举、错误码、领域实体和值对象 |

## 数据类与领域模型架构

### 1. 基础校验类

| 类/结构 | 作用 |
|---|---|
| `NumCheck` | 校验字符串是否为非零开头纯数字 ID |
| `DigitCheck` | 校验字符串位数是否符合要求 |
| `StringID` | 字符串 ID 抽象基类，定义统一 `SetValue()` 与取值接口 |
| `BookLocation` | 描述馆藏位置：分馆、楼层、区域、书架、层号，并提供有效性检查 |

### 2. ID 值对象

| 类 | 规则 | 用途 |
|---|---|---|
| `ISBN` | 10 位或 13 位纯数字，非零开头 | 图书 ISBN |
| `UserID` | 8 位纯数字，非零开头 | 普通读者账号 |
| `AdminID` | 5 位纯数字，非零开头 | 管理员账号 |
| `VolumeID` | 纯数字，非零开头 | 单册条码/编号 |

这些 ID 类将格式校验内聚在实体内部，避免业务层直接处理裸字符串。

### 3. 图书与馆藏实体

#### `Book`

`Book` 是书目级实体，字段包括：

- ISBN
- 书名
- 作者列表
- 出版社
- 图书分类
- 出版年份
- 出版语种
- 单册列表
- 图书简介
- 有效性标记 `IsValid`

#### `Volume`

`Volume` 是单册级实体，字段包括：

- 单册号 `VolID`
- 可用状态 `Availability`
- 是否开架 `IsOpenshelf`
- 馆藏位置 `BookLocation`
- 应还日期 `DueDate`
- 当前借阅者 `BorrowerID`
- 备注 `Note`
- 有效性标记 `IsValid`

一个 `Book` 可以包含多个 `Volume`，对应数据库中的一条图书记录与多条单册记录。

### 4. 借阅流水实体

#### `LoanRecord`

`LoanRecord` 描述一次单册借阅行为，字段包括：

- 自增流水号 `RecordID`
- ISBN
- 单册号
- 借阅者 ID
- 是否已归还
- 是否逾期
- 借阅日期
- 应还日期
- 归还日期

该实体是借阅历史、全局流水、高级检索和统计分析的基础。

### 5. 账户实体

#### `ReaderAccount`

普通读者账户字段包括：

- 8 位用户 ID
- 用户名
- 账户有效性
- 身份类型 `Auth::Reader`
- 密码盐 Salt
- 密码哈希 PasswordHash
- 借阅上限 BorrowLimit

#### `AdminAccount`

管理员账户字段包括：

- 5 位管理员 ID
- 固定用户名 `Admin`
- 账户有效性
- 身份类型 `Auth::Admin`
- 密码盐 Salt
- 密码哈希 PasswordHash
- 借阅上限固定为 0

### 6. 枚举体系

`TypeCode.h` 中定义了系统主要枚举：

| 枚举 | 说明 |
|---|---|
| `Availability` | 单册可用、外借中、丢失、加工中、已删除等状态 |
| `Library` | 北馆、西馆、经管馆、人文社科馆、法律馆等分馆 |
| `Area` | 东/西/南/北区 |
| `ErrorCode` | 用户错误、业务错误、系统/数据库错误等统一状态码 |
| `Language` | 中文、英文、俄文、法文、西文、阿文、日文、韩文、其他 |
| `Category` | 文学、哲学、语言学、艺术、技术 |
| `Auth` | 读者或管理员 |
| `Filter` | 借阅流水检索条件，如 ISBN、VolID、BorrowerID、日期等 |

## 数据库架构

系统使用 SQLite，并在初始化时自动创建以下表：

### 1. `Book` 图书表

保存书目级信息：ISBN、题名、简介、出版社、出版年份、语种、分类、有效性、创建时间、编辑时间。

### 2. `Authors` 作者表

保存图书作者列表：ISBN、作者姓名、作者顺序。该表与 `Book` 通过 ISBN 关联，支持一本书多作者。

### 3. `Volume` 单册表

保存单册信息：单册号、ISBN、备注、可用状态、是否开架、有效性。主键为 `(ISBN, VolId)`。

### 4. `VolLocation` 位置表

保存单册馆藏位置：分馆、楼层、区域、书架、层号。主键同样为 `(ISBN, VolId)`，与 `Volume` 一一对应。

### 5. `Account` 账户表

统一保存读者与管理员账户：ID、用户名、身份、有效性、借阅上限、盐、密码哈希、创建时间、编辑时间。

### 6. `LoanRecord` 借阅流水表

保存借阅行为：流水号、ISBN、单册号、借阅者、是否归还、是否逾期、借阅日期、应还日期、归还日期。

### 7. 索引设计

初始化时会创建出版社、出版年份、分类、作者名、馆藏分馆、账户用户名、账户身份、账户有效性、借阅日期、借阅单册、借阅者等索引，用于提升检索与统计性能。

## 核心业务流程

### 1. 启动流程

```text
main.cpp
  ├─ 创建 QGuiApplication
  ├─ 创建 DatabaseController
  ├─ initializeDatabase() 建库建表
  ├─ 注入默认管理员账号（如不存在）
  ├─ 创建 QQmlApplicationEngine
  └─ 加载 LibManager/Main.qml
```

### 2. 登录流程

```text
QML 输入账号密码
  └─ SystemBridge.loginReader/loginAdmin
       ├─ 构造 UserID/AdminID 并校验格式
       ├─ Login::UserLogin 查询账户
       ├─ PasswordCheck 校验盐化哈希
       ├─ 检查账户有效性与身份
       └─ 保存当前会话并通知 UI
```

### 3. 注册流程

```text
QML 输入读者 ID、用户名、密码、确认密码
  └─ SystemBridge.registerReader
       ├─ 校验 UserID 格式
       ├─ Register::UserRegister
       ├─ 检查账户是否已存在
       ├─ 检查密码长度和两次密码一致性
       ├─ 生成盐与哈希
       └─ AccountDAO.updateReaderInfo 写入账户
```

### 4. 借阅流程

```text
读者选择图书与单册
  └─ SystemBridge.borrowVolume
       ├─ 校验当前用户身份
       ├─ 查询图书与单册
       ├─ 检查单册是否可借
       ├─ 检查读者借阅上限与逾期情况
       ├─ 更新单册为外借状态
       └─ 写入 LoanRecord 借阅流水
```

### 5. 归还流程

```text
管理员选择 ISBN 与单册号
  └─ SystemBridge.returnVolume
       ├─ 检查管理员权限
       ├─ 查询图书与单册
       ├─ 查找未归还借阅流水
       ├─ 更新流水 ReturnDate / IsReturned / IsOverdue
       └─ 更新单册为可外借状态
```

## 错误码与异常处理

系统用 `ErrorCode` 统一表达结果状态：

- `0`：成功。
- `1000` 段：用户与账户错误，如用户不存在、密码错误、无权限、账户无效。
- `2000` 段：业务错误，如空输入、非法输入、单册已借、达到借阅上限、图书/单册不存在。
- `3000` 段：系统与数据库错误，如搜索失败、读写失败、数据库忙、事务失败。
- `9999`：其他错误。

数据库层使用 `DatabaseException` 同时携带业务错误码、Qt SQL 错误信息和可输出的字符串信息，便于向上层返回错误码并在日志中定位 SQL 问题。

## 系统优点

1. **分层较清晰**
   - QML、桥接层、业务层、DAO 层、数据实体层分工明确，便于理解和扩展。

2. **领域模型相对完整**
   - 图书、单册、位置、借阅记录、读者、管理员等实体划分清楚。
   - ID 类型使用专门类封装格式校验，减少错误输入进入业务流程。

3. **角色权限边界明确**
   - 普通读者和管理员使用不同 ID 类型与登录入口。
   - 图书维护、单册维护、读者管理、全局流水查询等功能限制为管理员。

4. **密码存储较安全**
   - 密码采用随机盐加 SHA3-512 哈希，不直接存储明文密码。

5. **数据库初始化自动化**
   - 首次启动即可自动创建表和索引，减少手动部署步骤。

6. **支持软删除与历史保留**
   - 图书、单册、账户包含有效性字段，有利于保留借阅历史关联。

7. **前端数据绑定友好**
   - `SystemBridge` 将 C++ 对象转换为 `QVariantList` 和 `QVariantMap`，适合 QML 表格、分页和详情视图绑定。

8. **具备统计分析扩展基础**
   - `Stats` 和 `LoanRecordDAO` 已封装热门图书、借阅达人、按时间统计等 OLAP 风格接口。

## 系统不足与改进方向

1. **业务服务以 `.cpp` 方式被桥接层包含**
   - `SystemBridge.cpp` 直接 `#include "src/UserService.cpp"` 和 `#include "src/LibraryService.cpp"`，不利于独立编译、单元测试和模块边界维护。建议拆分对应头文件，并在 CMake 中按源文件编译链接。

2. **部分输入校验还可增强**
   - ISBN 当前只校验长度和数字，未校验 ISBN-10 / ISBN-13 校验位。
   - 出版年份、楼层、书架、层号等字段仍可加入更严格范围校验。

3. **错误提示对用户不够友好**
   - 前端部分错误仍以错误码输出为主，建议建立错误码到中文提示的统一映射。

4. **默认管理员密码硬编码**
   - 默认账号便于演示，但正式部署应改为首次启动引导设置、环境变量注入或初始化脚本创建。

5. **缺少自动化测试**
   - 当前仓库未见专门的单元测试或集成测试目录。建议补充 ID 校验、DAO、登录注册、借还流程、统计查询等测试。

6. **数据库连接模型较简单**

## 构建与运行

### 前置条件

- 安装 Qt 6，且包含 Quick、QML、SQL、Widgets 模块。
- 安装 CMake 3.16 或更高版本。
- 编译器需支持项目使用的 C++ 标准。

### 使用 CMake 构建

```bash
cmake -S . -B build
cmake --build build
```

### 运行

构建成功后运行生成的可执行程序，例如：

```bash
./build/appLibManager
```

首次运行会在工作目录中创建或打开 `LibManager.db`。如数据库初始化失败，程序会在加载 QML 前退出。
